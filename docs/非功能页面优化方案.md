# 非功能页面优化方案

> 目标：把博客打磨到生产级、可交付。重点：**首页 / 文章详情页 / 所有页面的通用问题（字体、间距等）**。
> 约束：不大改基本布局；优先复用现有 token / 容器 / 卡片语法；浅色 + 深色主题都要成立。
> 落地时每处改动按 CLAUDE.md §3 做桌面 + 移动端实机检查。

## 0. 范围

**本次优化：** 首页、文章详情页、以及所有页面的通用问题（字体一致性、间距节奏、死代码）。
**本次不动：** Leaves 留言页、Topics 话题页。
**功能页面（始终不动）：** sports / photography / media / tools / travel / copyright。

---

## 1. 全站通用问题（最高优先级）

### 1.1 字体一致性

- 核对所有 in-scope 页面：正文/标题走 `--serif-font` 或 `--display-font`，导航/标签/日期/元信息走 `--ui-font` / `--field-font`，代码/数字走 `--code-font`。
- 重点排查 magazine 系列页面里普通正文被切成 UI 字体的地方是否合理（当前 ledger 骨架整体用 `--ui-font`，属于有意设定，保留；只修「同类信息字体不一致」的个例）。
- 不引入新字体家族。

### 1.2 字号 / 层级一致性

- 同类信息必须同层级：元信息统一落在 `0.68rem–0.78rem`，列表标题 `0.9rem–1.1rem`，不忽大忽小。
- 优先复用 `--text-md / --text-lg / --text-xl / --text-2xl`，不绕开 token 造新尺寸；确有需要的局部尺寸要能说明理由。

### 1.3 间距节奏一致性

- 优先用 spacing token `--space-2xs … --space-4xl`，清理脱离系统的 magic number。
- 相邻模块对齐 vertical rhythm；大区块间距沿用 `clamp()` + token 组合。

### 1.4 死代码 / 重复 style（跨页面）

多个页面叠了 2–3 个互相覆盖的 `<style>` 块，后写的靠源码顺序压前面，留下大量死代码。
详见 §4「代码清理计划」。**纯清理，合并前后逐页截图零变化比对。**

## 2. 首页 Home（`src/pages/index.astro`）

1. **最后一行列表项底部仍带 `border-bottom`**，与页脚间留下悬空分割线。
   → 给 `.home-latest-list .home-latest-row:last-child` 去掉底边框。
2. **日期分隔符不统一**：特色项 `MM /`、列表项 `MM.`。
   → 默认保留（特色 vs 列表的有意区分）；如需统一二选一，待确认再动。
3. 其余（特色大日期、标题 clamp、摘要两行截断）不变。

## 3. 文章详情页（`src/layouts/BlogPost.astro`）

1. **同页存在两大套 `<style is:global>`**：前一套（entry-sheet 卡片、code-wrapper 圆角、post-title `clamp(1.82rem,…,2.42rem)`）大部分被文件末尾 `.blog-post-layout` 那套完全覆盖（卡片去边框、标题放大到 `clamp(2rem,…,3.65rem)`、code-wrapper 去圆角）。属于死代码，按 §4 清理。
2. **正文排版核对**：正文 `font-size:1.03rem` + `line-height:1.86`，h2/h3/h4 的 margin 节奏，确认与全站阅读气质（`--lh-body` 1.82）一致，无忽松忽紧。
3. **元信息栏**：`post-kind / 日期 / min read / Updated` 字体统一为 `--field-font`/`--ui-font`，字号落在元信息层级区间。
4. **TOC 右栏**：确认 hover/active 反馈克制、字号一致。
5. 结构、卡片纸感、布局不改，仅收敛样式 + 核对排版。

## 4. 代码清理计划（死代码 / 重复 style）

原则：**只删被完全覆盖的历史样式，保留最终真正生效的一套；合并前后逐页截图必须零变化。**

| 文件 | 现状 | 处理 |
|------|------|------|
| `layouts/BlogPost.astro` | 顶部 `<style is:global>`（entry-sheet 卡片/圆角/小标题）+ 末尾 `<style is:global>`（.blog-post-layout 全面覆盖） | 合并为一套，删除被覆盖的卡片/圆角/旧标题字号等死规则 |
| `layouts/About.astro` | 3 个 style 块互相覆盖 + `.about-profile__rule`（DOM 存在但 `display:none`）+ 注释残留 | 合并为一套；移除 rule 死元素与注释残留 |
| `layouts/Message.astro` | 3 个 style 块互相覆盖（`min-height` 被 `:0` 覆盖等） | 合并为一套（本次不改留言页视觉，仅清理） |
| `pages/tags/index.astro` | 保留整套被覆盖的 card-cloud（渐变/动画/span 网格）死代码 | 删除死样式，保留 ledger 一套（本次不改话题页视觉，仅清理） |
| `pages/tags/[tag].astro` | 多个叠加 style 块 | 落地前先截图，合并为一套 |
| 各文件 `.sr-only` 重复定义 | 多处重复 | 可提到全局 `style.scss`，各页移除本地副本（低优先，收尾项） |

---

## 5. 落地顺序

1. 首页去尾部悬空分割线（收益直接、风险低）
2. 文章详情页 style 收敛 + 正文排版核对
3. 全站字体/字号/间距一致性核对（§1）
4. 代码清理（§4，逐页零变化比对）

## 6. 每步强制检查（对齐 CLAUDE.md §3）

- 起 dev server，实机看**桌面 + 移动端**（本方案基本都涉及布局/字号/换行/列表）。
- 核对：字体归位、同类同层级、正文行长、模块节奏、对齐线、卡片纸感、浅/深色主题、中英混排断行。
- style 合并后**逐页与合并前截图比对**，确认零视觉变化。
- 不以「代码正确」为完成标准；观感别扭优先修观感。
