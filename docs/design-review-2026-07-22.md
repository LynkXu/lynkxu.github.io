# 极简阅读版设计审查（2026-07-22）

> 范围：`redesign/minimal-reading` 分支的博客主体页（首页 / 归档 / 文章 / 标签 / 关于 / 碎语 / 留言 / 版权），不含专题页。
> 关注点：**视觉设计问题**——字号、间距、背景与配色、模块布局、组件样式、层级与一致性。
> 参考站（仅作方向校准，不照搬）：<https://ahdw.casacam.net/> —— 双语标题、按月分组+条数、右对齐日期、分类索引、极安静但结构清晰。
> 相关文件：`src/styles/reading.scss`、`src/layouts/ReadingShell.astro`、各页面 `.astro`。

本文只记录「设计层面」的问题与改进方向，不含已修复的内容 Bug（mastodon 噪音标签、摘要裸链接、长 URL 换行、文章尾部留白已在上一轮修掉）。

---

## 一、跨页面的核心问题（优先级最高）

这几条是「很多页面看起来有问题」的根因，建议先解决，其余页面问题大多会连带缓解。

### 1. 字号层级被压扁，缺乏对比 ⭐️
- 非文章页里，视觉最大的元素往往是页眉里的品牌 `LynkXu`（`clamp(1.7rem,3.4vw,2.05rem)` / 700），而正文区的信息几乎都挤在 `0.78rem`（`--r-text-xs`）到 `0.9rem`（`--r-text-sm`）之间的灰字。
- 结果：首页、标签、留言等页面读起来「一整片小灰字」，没有明确的主—次—辅层级，页面像半成品。
- 方向：为「页面标题 / 区块标题 / 列表标题 / 元信息」建立更清晰的四级对比；区块标题不要停留在 `xs` 的说明文字尺寸。

### 2. 配色接近纯灰阶，强调色几乎不出现 ⭐️
- 语义蓝 `--r-link: oklch(0.45 0.07 250)` 只在**正文超链接**里出现；列表标题、导航、日期实际用的是 `--r-ink / --r-ink-soft`，只有 hover 才变蓝。
- 结果：首页/归档/标签等索引页整屏是墨灰色，缺一个稳定的落点色，显得沉闷、缺乏「可知/理性但有生气」的气质。
- 方向：确定强调色的出现规则（例如日期、区块「全部」、当前导航、hover 统一走一个克制蓝），让灰阶里有一条一致的低饱和色线索。

### 3. 日期格式三套并存，不统一 ⭐️
- 首页碎语/文章列表：`YYYY.MM.DD`（如 `2026.07.08`）。
- 归档：`MM.DD`（如 `07.08`）。
- 碎语页：`YYYY.MM.DD · HH:mm`。
- 方向：统一一套日期展示规则（列表用一种、正文详情用一种），至少同类页面之间一致。

### 4. 分隔符/指示系统不统一
- 页眉主导航用「·」中点分隔（`nav a + a::before`），且当前项又叠加下划线——两套指示语言并存。
- 页脚专题链接却是纯间距分隔（无中点）。
- 标签详情页的「全部标签 · a · b …」又是一套行内中点。
- 方向：统一「导航/链接簇」的分隔与激活表达。

### 5. 交互控件存在感过低
- 点赞（爱心）在文章尾、碎语条目里几乎不可见（默认极淡，靠 hover 才出现）；搜索/主题是右上角两个 `0.78rem` 灰字；分页只有极小的「1 2」。
- 方向：这些控件不需要变重，但要给到最基本的可点击暗示（尺寸/对比/图标其一），尤其点赞当前近乎「隐身」。

### 6. 正文行距偏松，长文显稀
- 正文 `--r-text: 1.125rem`、`--r-lh-body: 1.9`、段距 `1.25em`、栏宽 `66ch`。
- 1.9 行距对 18px 中文衬线偏松，叠加窄栏与段距后，长文观感偏「空/散」。
- 方向：正文行距收敛到约 `1.75–1.8`，与 CLAUDE.md 基线（`--lh-body ≈ 1.8`）对齐。

## 二、逐页面设计问题

### 首页 `/`
1. **没有页面标题/锚点**：直接以极小灰字 `最近文章`（`.r-section__title`，`0.78rem` 且被设为 `<h1>`）开场，全页最大的字是页眉品牌，内容区没有视觉重心，像被截断的片段。参考站至少有一个明确的 H1。
2. **区块头太弱**：`最近文章` / `碎语` 与右侧「全部」都在 `xs` 灰字，区块标题读起来像图注而非小节标题。
3. **12 条文章是一条无分组的平铺梯子**：同字号同字重、`gap: 1rem`，没有年份或任何分组，和「归档」按年分组的做法不一致，信息密度低又缺节奏。
4. **日期列空档大**：日期列固定 `6.5rem` + `gap 1rem`，日期又是最淡的 `--r-ink-faint`，标题离日期很远，中间一段横向死白。
5. **碎语预览无悬挂缩进**：多行摘要换行后与日期左对齐，日期与正文的区分被抹平。

### 归档 `/blog`
6. 本页结构最好。但**年份标签偏弱**：`2026` 用 `text-sm` 灰字、无条数；相较大间距（`margin-bottom: 2.75rem`）容易被淹没。可考虑加条数、略提升年份标签的存在感。
7. 日期为 `MM.DD`，与首页不一致（见跨页问题 3）。

### 文章页（`BlogPost.astro`）
8. 正文行距偏松（见跨页问题 6）。
9. **引用块区分度低**：`blockquote` 仅左边线 + `ink-soft`，字号与正文一致，长引用与正文几乎同质。
10. **目录 `目录` 折叠**：`summary` 是 `xs` 灰字、无标记，可点击暗示弱。
11. **文末收束区偏空**：居中「·」+ 近乎隐形的点赞，无改动记录时该区域读起来空。已收紧间距，但点赞控件本身的存在感仍需处理（见跨页问题 5）。

### 标签索引 `/tags`
12. **「标签云」名不副实**：所有标签同字号，仅在标签后跟一个极小条数（如 `生活 10`）。没有大小/字重区分的「云」其实就是一行折行的行内链接，条数数字只增噪音、不建层级。
13. 折行的标签行（`ui-font` / `text-sm` / `gap .5rem 1rem`）扫读困难，看不出哪些是大主题。可考虑按条数排序并用字号/字重体现权重，或改成带条数的结构化列表（类似参考站分类索引）。

### 标签详情 `/tags/[tag]`
14. **标题下的「全部标签 · a · b …(12)」切换行噪音大**：一长串中点分隔的淡灰标签直接压在标题下方，和真正的文章列表抢注意力，且与 `/tags` 页内容重复。建议弱化或改为「返回全部标签」单个入口。

### 关于 `/about`
15. **折叠件层级混乱**：`联系`/`赞助` 是小节标签（图注风格），其下是 `WeChat/Alipay`、`Crypto` 两个 ▸ 折叠，页尾又有一个同级缩进的 `博客历程` ▸ 折叠——三处折叠件视觉层级/缩进不统一。
16. **头像偏小**：`56px` 圆头像配大号衬线 `Hi, I am Lynk`，头像显得次要、失衡。
17. traits（兴趣/运动/气质）标签列 `3rem` 对两字标签偏窄；这块上下细线是全页唯一的「卡片感」分隔，风格可，但需与其它小节的分隔语言协调。

### 碎语 `/shuoshuo`
18. **信息流缺锚点**：条目之间仅靠 `gap 2.75rem` 空白分隔，无细线/标记，日期时间又是最淡灰字，长列表读下来缺少定位点。
19. 媒体网格 `max-height 140–220px` 偏小，多图裁切很紧。
20. 分页「1 2」过于隐蔽（见跨页问题 5）。

### 留言 `/message`
21. **未加载时页面过空**：标题 `留言` + 一行「按需加载留言…」+ 一个小描边按钮，首屏几乎空白，像未完成。可加一句说明或轻框架，避免空场。

### 版权 `/copyright`
22. 结构清晰。仅小问题：`共享/改编/署名/非商业` 用**衬线粗体**（body 尺寸）作标签，比其上小节标题 `你可以做什么`（`xs` 灰字）还显重，标签与小节标题的轻重关系略倒挂。

## 三、页眉 / 页脚（ReadingShell）

23. **页眉重心失衡**：品牌 `LynkXu` 很大很重，右侧 `搜索 / 主题` 只有 `0.78rem` 淡字，像浮在右上角留白里的附注；可参考「简介·About」那样给右侧一个能与品牌相称的落点。
24. **页眉整块偏挤**：品牌 + 导航 + 底部细线挤在很小的垂直间距里（`gap .65rem`、`padding-bottom 1rem`），与舒展的正文形成「上紧下松」。
25. **两行导航语言不一致**：页眉主导航用中点、当前项下划线；页脚专题链接用纯间距、无激活态（见跨页问题 4）。
26. 导航项对比偏低：`--r-ink-soft` 文字 + `--r-ink-faint` 中点，安静到接近可读性下限。

## 四、可借鉴参考站的结构思路（不照搬视觉）

- 归档/标签**带条数**的分组（`2026-07 (1):`），比现有纯年份更有信息量。
- **右对齐、等宽数字**的日期列，让列表右缘形成整齐的时间轴。
- **分类/标签索引**做成结构化分组列表，而非无层级标签云。
- 允许**双语小标题**（如 `归档·Archive`）作为轻装饰与层级锚点——与本站「可知/理性」气质契合，但需控制不喧宾夺主。
- 一个**克制的分组切换控件**（参考站的 pill）——若做，需纳入本站按钮语言，避免变成异物。

---

## 五、建议的处理顺序

1. 先定**类型层级 + 强调色规则 + 日期格式**（跨页问题 1/2/3）——这三条改完，绝大多数页面观感会同步提升。
2. 再统一**导航/分隔/交互控件**的表达（跨页问题 4/5）。
3. 收敛**正文行距**（跨页问题 6）。
4. 最后处理**各页局部**：首页加标题与分组、标签页重构、关于页折叠层级、留言空场、标签详情切换行。

> 所有改动都应落在现有 `--r-*` token 与既有容器/卡片语法内；需要新增的层级/颜色，优先补进 token 而非页面级魔法值。

## 六、本轮已实施（2026-07-22）

方向校准：抓取参考站 `https://ahdw.casacam.net/` 的实际样式——`body` 背景为纯白 `#ffffff`、正文近黑中性灰、链接同为近黑。可见其「生气」来自**结构与层级**而非颜色。因此本轮**保持安静的近单色调**，把力气花在层级、结构与一致性，而不是引入品牌强调色。

- **背景对齐参考站**：亮色 `--r-bg` 由冷调近白改为纯白 `#ffffff`（含 `html` 兜底背景与 `[data-theme=light]`）；代码底色、分隔线的冷调收敛为中性，去掉与纯白冲突的偏蓝倾向。暗色主题维持原近黑。
- **类型层级**：区块小标题 `.r-section__title` 由 `xs/淡` 提为 `sm/600/ink-soft`，并给 `.r-section__head` 加底部细线；归档年份标题提为 `0.95rem/600` 并**带条数**（`2026 · 7`）+ 底部细线。
- **日期规则统一**：列表日期一律**右对齐、等宽数字**，形成右缘时间轴。规则：分组列表（归档，年份已在分组标题）用 `MM.DD`；非分组列表（首页最近、标签详情）用完整 `YYYY.MM.DD`。列表标题改为「标题在左、日期在右」。
- **分隔语言统一**：页眉导航去掉中点、改为纯间距 + 当前项下划线，与页脚一致；导航/工具字号提为 `sm`，颜色提为 `ink-soft`，可读性上来。
- **交互可见性**：修复点赞按钮——空态只在 `.post-end/.memo-item` hover 才显形，而重构后容器已改名为 `.r-article__tail/.r-memo`，导致永远隐藏；补上新容器的 hover 显形规则。碎语翻页字号提为 `sm` 并加顶部细线分隔。
- **页眉配重**：品牌略缩（`clamp(1.5rem,3vw,1.85rem)`），品牌与导航间距、页眉下外边距加大，缓解「上紧下松」。
- **首页**：两个区块有了带细线的小标题与右缘时间轴。
- **标签索引**：标签名提到正文字号（`ink`）、条数淡色等宽、间距加大，更可扫读。
- **标签详情**：移除冗长的「全部标签 · a · b …」切换行，改为安静的 `N 篇文章 · ← 全部标签` 元信息行。
- **关于**：头像 56→64px；`联系/赞助/博客历程` 折叠标记统一为同一 `▸` 语言。
- **留言**：标题下补一句引导语，避免首屏空场。
- **正文行距**：`--r-lh-body` 1.9→1.8，长文更紧凑但仍舒展。
